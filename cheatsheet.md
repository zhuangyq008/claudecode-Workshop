# 🚀 Claude Code SuperPowers 三步链 — Cheatsheet

直接复制下面的 prompt 到 Claude Code 里执行。

---

## 第一步：Explore（探索理解）

```
请帮我分析这个项目的整体结构和代码质量。具体要求：

1. 列出所有源代码文件和它们的职责
2. 找出所有安全风险（硬编码密钥、SQL 注入、权限问题等）
3. 找出所有代码质量问题（命名、重复代码、缺少文档等）
4. 给出一个优先级排序的修复建议清单

输出格式：用表格列出问题，包含文件名、行号、问题类型、严重程度、修复建议。
```

## 第二步：Plan（规划方案）

```
基于刚才的分析结果，请制定一个详细的重构计划：

1. 按优先级分批次：P0 安全问题 → P1 架构问题 → P2 代码质量
2. 每个批次列出具体要改哪些文件、怎么改
3. 考虑向后兼容性——API 接口不能变
4. 需要新增哪些文件（比如测试、配置模板）

输出一个可执行的 checklist，每项都是一个独立的、可验证的改动。
```

## 第三步：Execute（执行修复）

```
请按照刚才的重构计划，从 P0 安全问题开始逐项执行：

1. 先修复所有硬编码密钥 → 改用环境变量
2. 修复 SQL 注入 → 改用参数化查询
3. 添加认证中间件
4. 重命名所有不规范的变量名
5. 拆分过长函数
6. 消除重复代码
7. 添加 type hints 和 docstring
8. 写单元测试（至少覆盖核心逻辑）

每修复一项，简要说明改了什么、为什么这么改。
```

---

## 💡 进阶用法

### 单文件深度审查
```
请审查 dev-track/src/routes.py 的安全性，逐行分析潜在风险，并直接修复。
```

### 生成测试
```
请为 dev-track/src/ 下所有模块生成 pytest 单元测试，要求覆盖率 > 80%，包含正常路径和异常路径。
```

### 运维场景
```
请审查 ops-track/iam-policy-risky.json，列出所有违反最小权限原则的地方，并生成一个修复后的版本。
```
